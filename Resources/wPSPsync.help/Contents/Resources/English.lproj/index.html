wPSPsync Help

wPSPsync scans a PSP storage root and a sync root, compares save folders, and copies the newest or missing saves to the other side.

Important risk and warranty notice

wPSPsync copies, replaces, and restores save folders. A wrong root selection, interrupted copy, storage failure, software bug, or unexpected save layout can cause save data loss or overwrite newer saves.

The software is provided as is, without warranty of any kind. Automatic backups are a safety feature, not a guarantee.

Choose roots

Choose the top-level folder for both sides. The PSP storage root should contain PSP/SAVEDATA. The sync root is the folder that contains or will contain PSP/SAVEDATA.

If you accidentally choose PSP or SAVEDATA, wPSPsync normalizes the path while scanning and syncing.

Selected roots are saved and restored between launches.

Sync saves

Each row shows the PSP timestamp, sync timestamp, size, game title, and sync state.

  • Synced: both sides have the same latest save file time.
  • PSP newer: the PSP copy will be copied to the sync root.
  • Sync newer: the sync copy will be copied to PSP storage.
  • Only on PSP or Only in sync: the missing side will be created.

Use the checkboxes to choose what will be synced. The Sync button only affects checked rows.

Backups and restore

Backups are enabled by default. Before wPSPsync writes a new sync state, it creates a zip backup of the current sync root.

Backups are stored in ~/Library/Application Support/wPSPsync/Backups. The app keeps the five newest backups.

Use Restore Backup to replace the selected sync root with a saved backup. When backups are enabled, the current sync state is backed up before restore.

Disable Create backup before writing if you do not want automatic backups.

Risks and no warranty

wPSPsync copies, replaces, and restores save folders. A wrong root selection, interrupted copy, storage failure, software bug, or unexpected save layout can cause save data loss or overwrite newer saves.

Review the checked rows before syncing and keep separate backups of important saves.

Automatic backups are a safety feature, not a guarantee. Backups are stored on the same Mac and only the five newest backups are retained.

Game titles and artwork

wPSPsync includes a small game catalog and can import a JSON catalog. The app also reads local PSP save icons such as ICON0.PNG.

Enable Search SerialStation API to look up additional titles and artwork. Network lookup does not change save files.

Troubleshooting

If folders cannot be opened, choose the roots again from the sidebar so macOS grants wPSPsync access.

If a PSP is not detected automatically, choose the PSP storage root manually. Automatic detection requires a mounted volume that contains PSP/SAVEDATA.

If old accidental nested SAVEDATA folders exist from a previous version, remove them manually after confirming they do not contain needed saves.
